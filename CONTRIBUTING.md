# Contributing to SkyFlareInfra

Thank you for your interest in contributing to SkyFlareInfra!  
We welcome contributions from everyone, whether it's a small typo fix or a major feature.

---

## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** your fork:
   ```bash
   git clone https://github.com/<your-username>/SkyFlare.git
   ```
3. **Add the original repository as upstream**:
   ```bash
   git remote add upstream https://github.com/SkyFlareInfra/SkyFlare.git
   ```
4. **Create a new branch** for your change:
   ```bash
   git checkout -b my-feature-branch
   ```
5. **Make your changes** and commit:
   ```bash
   git add .
   git commit -m "Describe your changes briefly"
   ```
6. **Push to your fork**:
   ```bash
   git push origin my-feature-branch
   ```
7. Open a **Pull Request** against the `dev` branch of the SkyFlare repository.

---

## Code Guidelines

* Follow Go’s [standard formatting and style](https://golang.org/doc/effective_go).
* Keep commits small, focused, and descriptive.
* Write clear, maintainable code and include comments where necessary.
* Update or add tests for new code.
* Run all tests before submitting a PR.

---

## Reporting Issues

If you find a bug or have a suggestion for improvement:

* Check if it’s already reported under **Issues**.
* If not, open a new issue using the **Bug Report** template.
* Include steps to reproduce and relevant logs when possible.

---

## Pull Requests

When opening a Pull Request:

* Ensure it follows the structure of our PR template.
* Clearly describe the problem or feature.
* Make sure all checks and tests pass.
* PRs should target the `dev` branch unless otherwise specified.

---

## Code of Conduct

Please be respectful and constructive when interacting with others.
Our community values open collaboration, kindness, and professionalism.

---

Thanks again for contributing — your work helps make SkyFlareInfra better for everyone.
